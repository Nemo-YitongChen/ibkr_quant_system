# 2026-06-24 Supervisor Code Revision Health

## Context

Runtime artifacts on 2026-06-24 were generated by an active Supervisor process that did not include recently added dashboard/readiness fields. This can happen when the repository is updated but the long-running Supervisor process has not been restarted. The system needs to make this visible before relying on auto-order readiness artifacts.

## Changes

- Supervisor shutdown/running status now records `code_revision` from the current git `HEAD`.
- Dashboard now reads the current repository revision and compares it with the running Supervisor revision.
- Ops overview and Ops Health expose:
  - `supervisor_code_revision`
  - `dashboard_code_revision`
  - `supervisor_code_revision_status`
- Running Supervisor revision states:
  - `match`: running process and dashboard code match.
  - `missing`: running process is from an older build that does not report revision.
  - `mismatch`: running process reports a different revision from current dashboard code.
- `missing` produces a Supervisor warning; `mismatch` produces a degraded Supervisor health alert.

## Runtime Check

Regenerated dashboard with current code:

PYTHONDONTWRITEBYTECODE=1 python -m src.tools.generate_dashboard \
  --config config/supervisor.yaml \

- `supervisor_shutdown_reason=running_code_revision_missing`

This confirms the active Supervisor should be restarted before treating newly added auto-order readiness fields as authoritative.
